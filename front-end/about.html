Database Normalization

The schema is normalized up to 3NF — eliminating redundancy and ensuring data integrity.

1NF

First Normal Form

Every column holds atomic (indivisible) values. There are no repeating groups — ingredients are stored in a separate ingredient table, not as comma-separated values.

✓ All tables satisfy 1NF
2NF

Second Normal Form

Every non-key attribute is fully functionally dependent on the entire primary key. The junction table recipe_ingredient uses its own id PK, so quantity/unit fully depend on that PK only.

✓ No partial dependencies
3NF

Third Normal Form

No transitive dependencies exist. Ingredient details (name) live in ingredient, not duplicated in recipe_ingredient. User role (is_admin) is not derived from any non-key attribute.

✓ No transitive dependencies
BCNF

Boyce-Codd Normal Form

For every functional dependency X → Y, X is a superkey. All determinants in our tables are either primary keys or unique columns (e.g., ingredient.name is UNIQUE).

✓ All determinants are superkeys

Table Relationships

How the tables connect — in plain English and cardinality notation.

1 : M
recipe → recipe_ingredient  |  One recipe can have many recipe_ingredient rows. Each row represents one ingredient used with its quantity.
1 : M
ingredient → recipe_ingredient  |  One ingredient can appear in many recipe_ingredient rows — across different recipes.
M : M
recipe ↔ ingredient  |  Many-to-many relationship resolved via the recipe_ingredient junction table. A recipe has many ingredients; an ingredient is used in many recipes.
Independent
users is an independent entity managing authentication. Users have an is_admin flag to control write access. No FK to recipes — keeping auth decoupled from recipe data.
1 : M
recipe → ratings  |  One recipe can receive many ratings/reviews from different users.
1 : M
users → ratings  |  One user can write many reviews across different recipes.
M : M
users ↔ recipe (via bookmarks)  |  Many-to-many relationship: a user can bookmark many recipes, a recipe can be saved by many users.
